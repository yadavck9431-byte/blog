version controlling system is a software use to track code files and manage it. it provide ease, and rapid development to a large project where multiple coders work on a single project, facilitates a successful remote work too.

### problems for a mass developer :

lack of centralised workspace where ;

* tracking change in source code.
    
* tracking/control saved data of all version/history of the source code.
    
* track who have change and what he changed in the source code.
    
* reverting the error, instead of permanent deletion of code written after error.
    
* parallel work flow of multiple coders ona same project without interruption or overwriting the code.
    

### facilities :

* track of project and manage conflicts - track every changes in code files(of project); who had/have made changes, and what changes he made.
    

* data loss and recovery - save track of source code and changes made in it. it aloso provide the facility to revert to the particular version where error is made without deleting the code files written after that error.
